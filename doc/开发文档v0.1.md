pt + RAG（检索）+ Workflow 的收益远远大于 Fine-tuning。**

很多人误以为 AI 写小说靠模型，其实真正决定质量的是：

> **Workflow（工作流） > 数据 > Prompt > Model**

如果让我做，我甚至不会训练模型。

## **我还有一个建议，它会决定这个系统未来能不能做大。**

不要把每个 Skill 写成一个普通 Prompt，而是统一设计成一个**插件协议（Plugin Protocol）**。

例如每个 Skill 都约定：

```yaml
name: DialogueWriter

input:
  scene
  character
  style

output:
  dialogue
```

以后新增一个 `HorrorWriter`、`RomanceWriter`、`ForeshadowChecker`，只要遵循同一输入/输出协议，就能直接接入工作流，而不用改整个系统。

**如果这样设计，你做出来的就不是一个“AI 写小说工具”，而是一套真正可扩展的小说创作平台。**
