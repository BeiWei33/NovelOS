## What to build

修复新建章节（及人物/世界观/风格）失败后页面状态异常的问题。

当前问题：
1. 新建章节 API 调用失败时，只 `console.error`，不通知用户
2. 失败后可能导致 `loadAll()` 未完成，页面显示空白或仍在 loading
3. 后续 Tab 切换操作可能因加载异常而无法工作
4. 用户需要刷新页面才能恢复

需要：

1. **统一错误处理**：所有 CRUD 操作失败时显示错误消息（toast 或弹窗）
2. **状态恢复**：API 调用失败时，确保页面状态回退到操作前状态
3. **加载超时**：`loadAll()` 设置超时，超时后显示错误提示和重试按钮
4. **错误消息**：新建/删除操作失败时，显示具体错误原因（如"API 不可用"）

不在此 issue 范围内：
- 后端错误修复
- 其他 UI 改进

## Acceptance criteria

- [ ] 新建章节失败时显示错误 toast 而非静默失败
- [ ] 新建人物/世界观/风格失败时同样显示错误提示
- [ ] 失败后页面状态回退到操作前（不显示空白 loading）
- [ ] `loadAll()` 超时 10 秒后显示错误提示和重试按钮
- [ ] 删除操作失败时也显示错误提示
