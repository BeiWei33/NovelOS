## What to build

在章节详情页实现左侧导航树，替代当前简单的场景列表。

当前章节详情页左侧只是一个简单的场景列表按钮，没有层级感。

需要：

1. **导航树组件**：
   - 层级结构：Novel → Chapter → Scene
   - 当前小说名称在顶部
   - 章节可展开/折叠（显示该章节下的场景列表）
   - 场景可选中（点击后右侧编辑器加载对应场景）
   - 当前选中项高亮

2. **拖拽排序**（可选）：
   - 场景可拖拽重新排序
   - 章节可拖拽重新排序（较复杂，可延后）

3. **上下文菜单**：
   - 右键场景显示：编辑、删除、AI 写场景、润色
   - 右键章节显示：编辑、删除、一键生成整章

4. **响应式**：
   - 窄屏时导航树可折叠（汉堡菜单按钮）
   - 折叠后右侧编辑器占满全屏

设计决策需人类确认：
- 是否支持拖拽排序？
- 是否要显示场景的 planning.goal 作为导航标签？
- 是否需要章节缩略信息（摘要、评分）？

## Acceptance criteria

- [ ] 章节详情页左侧显示 Novel → Chapter → Scene 层级树
- [ ] 章节可展开/折叠
- [ ] 选中场景后右侧编辑器加载对应场景
- [ ] 窄屏时导航树可折叠（汉堡菜单按钮）
- [ ] 导航树章节项显示场景数量
