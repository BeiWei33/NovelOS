## What to build

修复完成后自动执行 Git commit 和 push，带有特殊标签便于追踪。

流程：
1. Patch 应用成功后，执行 `git add` 添加修改文件
2. 生成 commit message：
   ```
   auto-fix:修复 {error_type} 错误
   
   Error: {error_message}
   Fingerprint: {fingerprint}
   Patch-ID: {patch_id}
   
   Auto-generated by ErrorFixPipeline
   ```
3. Commit 带 `auto-fix` 标签（Git tag 或 commit message 标记）
4. Push 到远程仓库（可选，可通过配置开关）

安全措施：
- Commit 使用专用 Git identity（如 `NovelOS Auto-Fix <bot@novelos.dev>`）
- 不 push 到 `main` 分支，而是 push 到 `auto-fix/{patch_id}` 分支
- 可配置为仅 commit 不 push（本地验证后再手动 push）

不在此 issue 范围内：
- Patch 生成 —— #002
- 验证 —— #005
- 回滚 —— #004

## Acceptance criteria

- [ ] Patch 成功后自动执行 `git commit`
- [ ] Commit message 包含 error fingerprint、patch_id
- [ ] Commit 使用专用 Git identity
- [ ] 可配置是否自动 push
- [ ] Push 到独立分支 `auto-fix/{patch_id}`，不直接 push main
- [ ] Push 结果记录到 `auto_fix_log.push_status`
